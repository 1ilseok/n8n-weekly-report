## 📖 작성 가이드

### YAML Front Matter 필드 설명
- `date`: 작성일 (YYYY-MM-DD 형식)
- `day`: 요일
- `week_number`: 주차 (연도 기준)
- `author`: 작성자
- `trello_card_mapping`: 업무명과 Trello 카드 짧은 ID 매핑
  - 키: 업무/프로젝트 이름 (예: "Privacy-i MacOS Agent 자동화")
  - 값: Trello 카드의 짧은 ID (예: "VHYpqMEd")
  - 짧은 ID는 Trello URL에서 확인: `https://trello.com/c/VHYpqMEd/...`
  - AI가 업무 내용을 분석하여 자동으로 해당 카드에 댓글 추가
- `category`: 카테고리별 관련 카드 짧은 ID 그룹핑
  - 카테고리 이름과 관련 Trello 카드 짧은 ID 리스트
  - AI가 카테고리 정보를 활용하여 더 정확한 카드 선택
- `total_hours`: 총 작업 시간 (자동 계산 가능)
- `trello_synced`: Trello 동기화 여부 (n8n 자동화용)

### 업무 현황 테이블과 Trello 자동 동기화
- n8n 워크플로우(v3)에서 "업무 현황" 테이블의 모든 업무를 Trello에 자동 동기화
- 동작 방식:
  1. "업무 현황" 테이블의 모든 행을 파싱 (상태 무관)
  2. 각 업무의 "카테고리" 값으로 `category` 필드에서 관련 카드 ID 리스트 조회
  3. AI가 필터링된 카드 중 가장 적합한 카드 선택
  4. 선택된 카드에 상태, 업무 내용, 진도, 우선순위 등을 댓글로 추가
  5. 카드를 무조건 "진행 중" List로 이동
- 카드 짧은 ID는 Trello URL에서 확인: `https://trello.com/c/{shortLink}/...`
  - 예: `https://trello.com/c/VHYpqMEd/9-privacy-i-macos-agent-자동화`
  - 짧은 ID는 `/c/` 다음의 `VHYpqMEd` 부분
