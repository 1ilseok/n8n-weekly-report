## 📖 작성 가이드

### YAML Front Matter 필드 설명
- `date`: 작성일 (YYYY-MM-DD 형식)
- `day`: 요일
- `week_number`: 주차 (연도 기준)
- `author`: 작성자
- `trello_card_mapping`: 카테고리별 Trello 카드 ID 매핑
  - 카테고리 이름과 해당 Trello 카드 ID를 입력
  - n8n이 자동으로 해당 카드에 업무 내용 추가
- `total_hours`: 총 작업 시간 (자동 계산 가능)
- `trello_synced`: Trello 동기화 여부 (n8n 자동화용)

### 카테고리와 Trello 자동 매핑
- 테이블의 **카테고리** 값에 따라 자동으로 해당 Trello 카드에 내용 추가
- `trello_card_mapping`에 정의된 카테고리만 Trello에 동기화됨
- 카드 ID는 Trello URL에서 확인: `trello.com/c/{CARD_ID}/...`

### 카테고리 분류 기준
| 카테고리 | 설명 | Trello 매핑 |
|----------|------|-------------|
| Redmine | 일감, 버그, 기능요청, 피드백 | ✅ 자동 매핑 |
| Server-i | TestComplete, TC, Server-i | 자동화 카테고리에 포함 |
| WebKeeper | Playwright, WebKeeper | 자동화 카테고리에 포함 |
| 자동화 | PV, macOS, DB Script, 자동화 전반 | ✅ 자동 매핑 |
| 인프라 | Jenkins, Ansible, Grafana, NAS, Allure | ✅ 자동 매핑 |
| 문서화 | 가이드, 문서, README, Wiki | 기타 카테고리에 포함 |
| 회의 | 회의, 미팅, 리뷰, 협의 | ✅ 자동 매핑 |
| 학습 | 학습, 스터디, MCP, 신기술 | ✅ 자동 매핑 |
| 기타 | 위에 해당하지 않는 모든 업무 | ✅ 자동 매핑 |

### 난이도 기준
- **하**: 단순 반복 작업, 1시간 이내
- **중**: 일반적인 개발/테스트 작업, 2-4시간
- **상**: 복잡한 문제 해결, 새로운 기술 학습, 4시간 이상
